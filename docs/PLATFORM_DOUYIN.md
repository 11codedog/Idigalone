# 抖音平台接入计划

## 目标

隔离平台 API，让 Cocos 预览、抖音开发者工具、真机都能运行同一套玩法代码。

## 平台实现

| 实现 | 运行环境 | 职责 |
|---|---|---|
| `MockPlatform` | Cocos 预览和本地测试 | 内存存储、日志输出、假的分享和广告结果。 |
| `DouyinPlatform` | 抖音开发者工具和真机 | 用安全 Promise 和兜底逻辑封装 `tt.*`。 |
| `PlatformManager` | 所有环境 | 选择平台实现，并向业务层暴露稳定方法。 |

## 能力清单

| 能力 | MVP 行为 | 失败行为 |
|---|---|---|
| 登录 | 可用时尝试平台登录。 | 继续游客流程。 |
| 本地存储 | 保存金币、升级、设置、版本。 | 回退到内存并记录警告。 |
| 分享 | 可用时触发分享入口。 | 返回失败结果，不让游戏崩溃。 |
| 激励视频 | 为复活、补氧、双倍奖励预留接口。 | 返回不可用，走普通流程。 |
| 震动 | 挖矿、按钮、奖励时提供轻反馈。 | 不支持时静默忽略。 |
| 系统信息 | 可用时读取设备和平台信息。 | 使用保守默认值。 |

## 业务规则

- 玩法只调用 `PlatformManager`。
- `tt.*` 只允许出现在 `DouyinPlatform` 中。
- 广告在 MVP 阶段是可选能力，不能阻塞挖矿、结算、升级。
- 平台不可用路径必须能在 Cocos 预览中测试。

## 开发者工具测试

- 构建能在抖音开发者工具打开。
- 登录不可用时不启动崩溃。
- 存档读写能保存进度。
- 分享调用返回受控结果。
- 激励视频不可用时能回到游戏流程。
- 震动调用不会抛错。

## 真机测试

| 设备 | 必测项 |
|---|---|
| Android | 启动、下矿、存储、震动、后台返回。 |
| iOS | 启动、下矿、存储、震动兜底、安全区显示。 |

把设备型号、系统版本、构建版本、阻塞问题记录到 `docs/RELEASE_CHECKLIST.md`。

## 当前实现状态

- 已新增 `DouyinPlatform`，仅该文件允许封装 `tt.*`。
- `PlatformManager` 会在检测到 `tt` 时使用 `DouyinPlatform`，否则使用 `MockPlatform`。
- 已实现登录、本地存储、分享、激励视频、震动、系统信息的安全包装。
- 激励视频广告位 ID 暂时由调用方通过 `placement` 传入；正式接入前需要替换为抖音后台配置的真实广告位。
