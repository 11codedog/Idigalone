# 路线图

## 版本目标

| 版本 | 目标 | 验收标准 |
|---|---|---|
| v0.1 | 可玩的挖矿闭环 | 能开始、挖矿、收集、氧气压迫、卖矿、升级、重开。 |
| v0.2 | 完整单局 UI 闭环 | 首页、HUD、暂停、局内奖励选择、结算、升级界面可走通。 |
| v0.3 | 抖音可运行试玩版 | 平台适配、存储、分享、震动、激励视频占位、开发者工具可打开。 |
| v1.0 | 可提审版本 | 包体、性能、合规、弱网、真机检查完成。 |

## 阶段计划

| 阶段 | 目标 | 主要工作 | 产出 |
|---|---|---|---|
| 1. 项目骨架 | 让开发可控 | 规则、文档、目录、平台接口 | 项目文档和架构基线 |
| 2. 可玩原型 | 30 秒内验证乐趣 | 玩家、连续地形挖矿、矿石、氧气、背包、结算 | Cocos 预览 v0.1 |
| 3. UI 闭环 | 跑完一局 | 首页、HUD、暂停、结算、升级入口 | 首页 -> 游戏 -> 结算 -> 再开 |
| 4. 数据闭环 | 保存进度 | 存档、金币、升级、基础配置 | 退出重进后进度还在 |
| 5. Roguelite 层 | 增加每局变化 | 收集里程碑触发随机 3 选 1 增益、临时局内修正 | 每局手感不同 |
| 6. 抖音接入 | 进入平台工具链 | `DouyinPlatform`、存储、分享、震动、广告兜底 | 抖音开发者工具可打开 |
| 7. 体验打磨 | 提升留存感 | 反馈、音效、震动、节奏、难度 | 可给朋友试玩 |
| 8. 包体提审 | 准备审核 | 压缩、包体检查、隐私、权限、真机测试 | 提审检查表完成 |

## 第一周计划

| 天 | 工作 | 验收标准 |
|---:|---|---|
| Day 1 | 项目规则、文档、平台接口、`MockPlatform` | Cocos 预览没有 TypeScript 错误。 |
| Day 2 | 核心挖矿原型 | 玩家能挖矿、捡矿、消耗氧气并结束一局。 |
| Day 3 | 首页、HUD、结算、再来一局 | 能开始并重复完整单局流程。 |
| Day 4 | 存档、金币经济、升级 | 金币和升级退出重进后仍保留。 |
| Day 5 | Roguelite 局内奖励选择 | 玩家收集达标后能 3 选 1，并感受到局内效果。 |
| Day 6 | 抖音平台基础 | 开发者工具能打开，平台能力不可用时不崩。 |
| Day 7 | 真机和第一轮打磨 | 记录至少一台 Android 或 iOS 真机测试，可产出试玩包。 |

## 开发规则

每次 Codex 任务只改一个模块，并输出改动文件、Cocos 编辑器挂载说明和测试步骤。
