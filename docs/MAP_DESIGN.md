# 地图设计

## 当前目标

第一版地图重设计只解决一个核心问题：去掉左右边界，让矿洞横向可以无限延伸，但不增加复杂玩法事件。

本阶段不做：

- 废弃矿道。
- 大型空洞。
- 生物群落。
- 多层地图入口。
- 地图小地图或标记系统。

## 坐标规则

| 轴 | 规则 |
|---|---|
| `x` | 无限延伸，可以为负数，也可以是很大的正数 |
| `y` | `0m` 是地表，向下递增 |
| 横向边界 | 不存在，玩家可以一直向左或向右探索 |
| 纵向边界 | 暂时保留 `generatedDepth`，避免原型阶段生成无限深度 |
| 初始位置 | `(0, 0)` |

`MineGrid.width` 只表示当前 UI 默认显示列数，不再表示地图宽度。

## 生成规则

地图按坐标即时生成，不再依赖“先查哪个格子”的随机调用顺序。

```text
玩家需要查看某个格子
  -> 根据 seed + x + y + salt 算出确定性随机值
  -> 判断该坐标是否在矿脉中
  -> 如果是矿脉，按矿脉概率生成矿物
  -> 否则按深度生成背景矿、氧气、石头或泥土
  -> 玩家挖过的格子写入缓存
```

这样可以保证：

- 同一局内同一坐标永远生成同一种初始矿块。
- 查询顺序不会改变地图结果。
- 地图横向无限，但只缓存玩家实际看过或挖过的格子。
- 适合抖音小游戏的性能和包体约束。

## 矿脉规则

第一版矿脉按区块生成：

| 项目 | 当前规则 |
|---|---|
| 区块宽度 | 12 格 |
| 区块高度 | 8 米 |
| 矿脉形状 | 小型椭圆矿团 |
| 矿脉中心 | 根据区块坐标确定性生成 |
| 矿物类型 | 仍按深度矿物池和权重选择 |
| 富矿嗅觉 | 提高矿脉出现率和背景矿概率 |

设计意图：

- 玩家看到矿脉边缘后，可以选择横向追矿。
- 横向追矿会消耗氧气，所以不是无成本刷矿。
- 深度仍然决定高价值矿解锁，避免玩家只靠横向无限刷新跳过成长。

## 视口规则

`MineGridView` 不再固定显示 `x = 0..width-1`。

```text
显示列数 = RUN_CONFIG.gridWidth
玩家所在 x 尽量位于中间列
屏幕左侧世界坐标 = 玩家 x - floor(显示列数 / 2)
```

玩家向左或向右移动时：

- 世界坐标真实变化。
- 画面显示区域跟随玩家平移。
- 没有撞墙反馈。

## 氧气和风险

去掉左右边界后，必须继续保留资源压力：

- `0m` 地表左右移动不消耗氧气。
- 地下横向移动仍消耗氧气。
- 地下绕路、追矿、回程都要付出氧气成本。
- 氧气囊只提供短期救命机会，不应该让无限横向变成无风险探索。

## 后续扩展

后续地图体验可以按这个顺序加：

1. 斜向矿脉，让矿脉更像“地质结构”。
2. 小型空洞，制造绕路和回程路线。
3. 氧气囊群，形成短暂安全点。
4. 硬石带，制造路径选择。
5. 废弃矿道，作为稀有兴趣点。
6. 深度地质层视觉差异。

每次扩展都要保持一个原则：深度是收益主轴，横向是选择和风险，不是无限刷资源捷径。
