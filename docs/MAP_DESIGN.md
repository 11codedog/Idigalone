# 地图设计

## 当前目标

当前地图重设计解决两个核心问题：去掉明显格子挖掘，让玩家用 360° 摇杆在连续土层里自由掏洞；同时保留横向无限延伸和短局风险压力。

本阶段不做：

- 废弃矿道。
- 大型空洞。
- 生物群落。
- 多层地图入口。
- 地图小地图或标记系统。

## 坐标规则

| 轴 | 规则 |
|---|---|
| `x` | 连续坐标，无限延伸，可以为负数，也可以是很大的正数 |
| `y` | 连续坐标，`0m` 是地表，向下递增 |
| 横向边界 | 不存在，玩家可以一直向左或向右探索 |
| 纵向边界 | 暂时保留 `generatedDepth`，避免原型阶段生成无限深度 |
| 初始位置 | `(0, 0)`，后续位置可以是小数 |

旧 `MineGrid.width` 只保留给旧格子测试路径；当前运行流使用连续地形视口，不再用格子宽度表示地图宽度。

## 生成规则

地图按连续坐标采样即时生成，不再依赖“先查哪个格子”的随机调用顺序。

```text
玩家需要查看某个连续坐标
  -> 转成内部采样点
  -> 根据 seed + sampleX + sampleY + salt 算出确定性随机值
  -> 按深度生成空气、泥土、石头、氧气或矿物
  -> 玩家挖过的采样点写入地形 delta
```

这样可以保证：

- 同一局内同一采样点永远生成同一种初始材料。
- 查询顺序不会改变地图结果。
- 地图横向无限，但自然生成采样点不进长期缓存，只记录玩家挖过的地形 delta。
- 适合抖音小游戏的性能和包体约束。

## 缓存规则

横向无限地图必须避免缓存无限增长。

| 缓存 | 内容 | 上限 | 清理策略 |
|---|---|---:|---|
| `materialOverrides` | 玩家挖空后的采样点 delta | 后续按真机表现设置 | 超过上限时按距离或 chunk 清理 |
| 旧 `tiles` / `veinsByChunk` | 旧格子路径测试和回退 | 10000 / 4096 | 后续删除旧路径时一起移除 |

自然生成但未被玩家改变的采样点可以由坐标重新计算，所以不需要长期缓存。

被挖过的采样点只保证在 delta 保留范围内回溯一致；极端横向探索超过上限后，最远处修改记录可能被清理并按自然生成重新计算。后续如果要支持超长回程路线，需要新增 chunk diff 或压缩路径存储任务。

哈希规则使用无符号 32 位混合，负数坐标会先转成稳定的 32 位表示，再参与黄金比例常量和 avalanche 混合，避免负数区域生成分布明显偏斜。

## 矿脉规则

当前连续地形先按深度和确定性噪声生成材料，矿脉表现后续再从“单点矿物概率”升级为“连续矿带”。

| 项目 | 当前规则 |
|---|---|
| 内部采样尺寸 | `TERRAIN_CONFIG.sampleSize`，当前为 0.5m |
| 地形形态 | 连续采样点组成土层/岩层/矿物分布 |
| 挖掘形态 | 圆形刷子清除范围内采样点 |
| 地形 delta | `ContinuousTerrain.applyDigDelta()` 统一落地 |
| 矿物类型 | 仍按深度矿物池和权重选择 |
| 矿石吸附 | 玩家周围约 0.8m 内的矿石会自动进入背包，并把对应采样点清成空气 |
| 富矿嗅觉 | 后续接入连续地形概率修饰，不由 UI 特判 |
| 浅层矿物概率 | 当前基础概率约 1.5%，地表附近保持稀疏 |
| 峰值矿物概率 | 当前在约 140m 达到 20% 峰值，之后保持峰值而不是继续膨胀 |
| 可视矿石上限 | 同屏最多渲染 140 个矿石 sprite，按玩家位置附近优先，而不是按视口上沿优先 |

设计意图：

- 玩家看到矿物颜色后，可以选择横向、斜向或向上追矿。
- 自由追矿会消耗氧气，所以不是无成本刷矿。
- 小范围吸附用于减少视觉上贴到矿石却漏捡的挫败感；它不能吸氧气包、泥土或石头，也不能绕过背包容量。
- 深度仍然决定高价值矿解锁，避免玩家只靠横向无限刷新跳过成长。

## 矿物层级

| 深度 | 新出现矿物 | 视觉目的 |
|---:|---|---|
| 1m | 铜矿 | 建立基础采集反馈 |
| 8m | 煤矿 | 早期加入深色矿块，避免前 10 米单调 |
| 16m | 锡矿 | 加入浅亮金属色，提示矿层变化 |
| 28m | 铁矿 | 硬度开始变化 |
| 42m | 银矿 | 中层高亮奖励 |
| 65m | 金矿 | 明确高价值目标 |
| 82m | 翡翠矿 | 富矿层前段目标 |
| 105m | 水晶 | 富矿层高亮目标 |
| 135m | 红宝矿 | 深层高价值目标 |
| 165m | 黑曜矿 | 深层高风险目标 |

当前连续地形视觉按三层实现：

- `ContinuousTerrain` / `ContinuousRunManager` 使用连续坐标和地形采样，保证挖掘、背包、收益、奖励触发规则稳定。
- `TerrainVisualSampler.sampleSoilTiles()` 从真实地形样本生成土层 tile，已经挖成 `air` 的采样点不能再被画回土。
- `TerrainDigMask` 只负责当前挖掘路径的柔边和连续笔触表现，不能作为地形是否存在的唯一依据。
- `ContinuousTerrainView` 先画洞穴底色，再画真实土层，再画当前挖掘柔边，最后画矿石 sprite 和玩家。
- 矿石 sprite 使用 `assets/resources/art/sprites/ore_*.png`，同屏数量受控时必须围绕玩家附近优先，避免下半屏矿石被上沿采样顺序挤掉。
- 玩家 sprite 尺寸接近连续挖掘半径对应的屏幕直径，不能明显大于可通行/挖掘体积。
- 正式贴图后仍保持 `TILE_CONFIG` / `ORE_TYPES` 作为唯一矿物数据入口，不在 UI 中新增玩法规则。

## 视口规则

`ContinuousTerrainView` 浅层会保留地表可见区域；深度超过起始跟随阈值后，以玩家连续坐标为视口中心。

```text
浅层视口中心 = max(玩家深度, 地表 + 起始可见比例)
深层视口中心 = 玩家连续坐标
屏幕像素 -> 世界连续坐标
世界连续坐标 -> terrain.sample()
terrain.sampleAtCoordinate() -> TerrainVisualSampler 生成真实土层 tile 和矿石 sprite
ContinuousTerrainView -> 按层绘制
```

玩家向左或向右移动时：

- 世界坐标真实连续变化。
- 画面显示区域跟随玩家平移。
- 没有左右撞墙反馈。
- 长按输入按 `deltaTime` 推进，挖掘距离由时间、方向强度、硬度和氧气共同决定。
- 任何可视上限、裁剪、缓存或降采样都不能按“遍历顺序”决定重要内容；矿石、玩家附近洞口和当前挖掘路径必须优先保留。

## 氧气和风险

去掉左右边界后，必须继续保留资源压力：

- `0m` 地表移动不消耗氧气。
- 地下任意方向移动和挖掘仍消耗氧气。
- 地下绕路、追矿、向上回挖和回程都要付出氧气成本。
- 氧气囊只提供短期救命机会，不应该让无限横向变成无风险探索。

## 后续扩展

后续地图体验可以按这个顺序加：

1. 已完成：斜向矿脉，让矿脉更像“地质结构”。
2. 小型空洞，制造绕路和回程路线。
3. 氧气囊群，形成短暂安全点。
4. 硬石带，制造路径选择。
5. 废弃矿道，作为稀有兴趣点。
6. 深度地质层视觉差异。

每次扩展都要保持一个原则：深度是收益主轴，横向是选择和风险，不是无限刷资源捷径。
