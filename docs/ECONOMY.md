# 矿脉经济系统设计

## 设计前提

本项目面向抖音小游戏，不按传统长线 RPG 或买断制矿洞游戏设计经济。经济系统必须服务于短局循环、即时反馈和轻量成长。

| 抖音小游戏习惯 | 对经济设计的要求 |
|---|---|
| 单局时间短 | 30-90 秒内看到收益变化 |
| 竖屏轻操作 | 收益判断直观，不依赖复杂菜单 |
| 强反馈 | 挖到高价值矿、追到矿脉、结算增长要明显 |
| 低理解成本 | 矿物价值、背包压力、氧气风险一眼能懂 |
| 轻成长 | 每局都推动一点升级目标 |

## 核心目标

- 让玩家形成“再往下挖一点可能更赚”的冲动。
- 让玩家看到矿脉后产生“要不要横向追矿”的选择。
- 让背包、氧气、硬度、深度共同形成取舍。
- 让后续新增矿物主要改配置，不改核心采集和结算逻辑。

## 深度层级

| 层级 | 深度 | 主要矿物 | 体验目标 |
|---|---:|---|---|
| 地表浅层 | 1-30m | 铜矿、铁矿 | 教学、安全、稳定小收益 |
| 稳定矿层 | 31-80m | 铁矿、银矿、金矿 | 路线选择和背包压力开始出现 |
| 富矿层 | 81-160m | 金矿、水晶、黑曜石前置 | 收益明显提升，氧气压力变强 |
| 危险矿层 | 161m+ | 黑曜石 | 高硬度高价值，要求装备成长 |

## 当前矿脉生成规则

第一版已从“固定宽度地图”改为“横向无限地图”。

```text
玩家查看某坐标
  -> MineGrid 根据 seed + x + y 生成稳定随机值
  -> 按 12x8 的区块尝试生成小型椭圆矿脉
  -> 如果坐标落在矿脉中，按矿脉 richness 判定矿物
  -> 否则按深度判定背景矿、氧气、石头或泥土
  -> 玩家挖过的格子写入 tiles 缓存
```

约束：

- `x` 可以无限延伸，`MineGrid.width` 只表示 UI 显示列数。
- 深度决定可出现矿物池，横向不能跳过深度成长。
- `富矿嗅觉` 只提升矿脉/背景矿概率，不直接改 UI 或结算。
- 高价值矿更多以小片矿脉出现，不满屏铺开。
- 每一局开始时重置矿洞，避免上一局挖空状态污染新局。

## 结算公式

第一版保持可解释：

```text
矿石收入 = sum(矿石数量 * 矿石单价)
深度基础奖励 = floor(最大深度 / 20) * 6
装备倍率 = 1 + 矿石手册等级加成
局内增益 = 高价出货、深层奖金等 Roguelite 增益
最终收入 = floor((矿石收入 + 深度基础奖励) * 装备倍率) + 局内额外奖金
```

限制：

- 深度奖励只能是补充，不应超过主要矿石收入。
- 结算页必须展示收益来源，不能只显示总数。
- 装备倍率温和增长，避免数值膨胀过快。

## UI 展示

HUD 不展示全部矿物，避免挤爆顶部 20% UI 区。

HUD 显示：

```text
深度 / 氧气 / 背包 / 金币预估 / 当前增益 / 矿石总数与种类数
```

结算页展示：

```text
铜矿 x8 = 40
铁矿 x3 = 33
金矿 x2 = 70
矿石压缩：节省 4 格背包
深度基础奖励：12
本局收入：155
```

## 已落地代码约定

| 能力 | 落点 |
|---|---|
| 矿物类型 | `OreType`、`RunInventory` |
| 矿物配置 | `assets/scripts/config/GameConfig.ts` 的 `ORE_TYPES`、`ORE_DEPTH_CONFIG`、`TILE_CONFIG` |
| 采集结算 | `TileEffectResolver` 返回 `inventoryDelta`，`RunManager` 统一写入状态 |
| 矿洞生成 | `MineGrid` 按坐标确定性生成无限横向地图 |
| 视口显示 | `MineGridView` 以玩家为中心显示附近列 |
| HUD | 只显示矿石总数和种类数 |
| 结算页 | 按 `ORE_TYPES` 自动生成矿石收益明细 |

## 暂不做

- 市场价格每日波动。
- 多货币系统。
- 矿物合成。
- 复杂装备词条。
- 离线收益。
- 广告翻倍作为核心经济。
- 强制分享或强社交任务。
