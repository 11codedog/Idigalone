# 屏幕适配说明

## 目标

- 游戏按竖屏手机优先设计，基础设计稿尺寸为 `720 x 1280`。
- 构建到抖音小游戏后，主画面必须自动适配不同手机屏幕，不能缩成居中的小块。
- 背景允许裁切，但核心 HUD、连续矿洞主体、按钮和日志必须保持在可视区域内。

## 实现规则

- 分辨率策略统一由 `assets/scripts/ui/ScreenLayout.ts` 管理。
- `MiningDebugPanel` 只创建一个 `UiFactory`，后续界面不直接操作 Cocos 画布尺寸。
- `UiFactory` 使用两层结构：
  - 外层根节点跟随当前可视区域。
  - `DesignRoot` 保持 `720 x 1280` 设计坐标，并按屏幕尺寸缩放。
- 背景图使用 cover 策略：优先铺满屏幕，允许边缘裁切。
- 交互层不能只依赖整层缩放；运行中界面必须通过 `RunScreenLayout` 重新计算 HUD、矿洞材质、按钮和日志位置。
- 运行中界面采用 `20% 顶部 UI 区 + 80% 游戏主体区`：上方约 20% 放 HUD、提示和图例；下方约 80% 留给连续矿洞主体。
- 矿洞主体不显示左侧深度刻度，避免挤占主体区域宽度。
- 矿洞视觉不再显示明显格子边框；连续地形由像素材质表现，玩家图标作为更大的 overlay 展示。
- 运行中矿洞视觉默认使用至少 `160 x 220` 的采样结果，地形颗粒应明显小于玩家和矿物提示。
- 游戏主体外围不再绘制整块黑色背景，只保留背景图和矿洞材质本身；HUD 可以有自己的半透明面板。
- 普通 UI 不直接读取 `view.getVisibleSize()`，需要屏幕尺寸时从 `UiFactory.getLayoutMetrics()` 获取。

## 手机摇杆

- 运行中触摸任意位置会显示浮动摇杆，摇杆中心就是触摸起点。
- 手指不松开时，拖拽方向输出 360° 输入向量，支持斜向和向上挖掘。
- Cocos 预览中鼠标按下并拖拽也走同一套 360° 摇杆状态，方便不接真机时验证角度。
- 拖拽距离小于 deadzone 不移动，防止误触。
- 跨过 deadzone 后每帧按 `deltaTime` 推进，长按时间越久挖得越远。
- 松手或切出运行中状态时，摇杆隐藏并停止移动。
- 摇杆只负责输入转换，不计算挖掘、氧气、背包或奖励。

## 后续检查

- 手机真机截图中，主画面宽度应接近屏幕宽度。
- 横向黑边和纵向大黑边都不应出现。
- 抖音右上角胶囊区域不能遮挡关键操作按钮。
- 如果新增独立界面，必须复用 `UiFactory` 和 `ScreenLayout`。
- Cocos 浏览器预览如果选择 `1280x720` 横屏尺寸，界面会按高度缩小以完整显示；手机验收仍以竖屏真机或 `720x1280` 预览为准。

## 验证节奏

- 小改玩法和普通 UI：先用 Cocos 预览快速验证。
- 改屏幕适配、触摸输入、平台能力、资源、场景或构建配置：需要重新构建到抖音开发者工具，再用手机真机预览验证。
- 每次完成这类改动后，Codex 需要在总结里提醒是否必须重新构建，以及建议在哪个环境验证。
